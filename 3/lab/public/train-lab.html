  • 协方差：计算数据的协方差矩阵

    Cov=1n−1XTX

  • 选择主成分：选取前 k 个主成分，即前 k 个最大特征值对应的特征向量

W=[eignV1,eignV2,...,eignVk]

  • 投影：用选定的主成分矩阵 W 变换原始数据，得到降维后的数据。

    X′=XW

Softmax 激活函数

Softmax 是一种常用的激活函数，通常用于多分类问题的神经网络输出层。它的主要作用是将输出的多个值转换成一个概率分布，使得每个输出值都代表了该类别的概率。

原理：

  • 给定一个包含多个实数值的向量（例如神经网络的输出），Softmax 将这些值转换为一个概率分布。

  • 输出的每个值都在 [0, 1] 之间，并且所有输出的和为1，这样可以看作是每个类别的概率。

  • Softmax 函数的计算公式是：

    Softmax(zi)=ezi∑j=1nezj

    其中，zi 是每个类别的原始输出值，ezi 是其指数，分母是所有类别的指数和，保证了所有输出的概率之和为 1。

特点：

  1. 输出为概率：Softmax 输出的每个值表示每个类别的概率，且所有概率之和为 1。

  2. 适用于多分类问题：特别适合用于多分类的神经网络模型，通常应用在最后一层，用于输出模型对每个类别的预测概率。

  3. 放大差异：Softmax 函数会放大输入值之间的差异，如果某个类别的原始输出值远大于其他类别，它的概率将接近 1，其他类别的概率则接近 0。

 

Adma 优化器

简介：

Adam（Adaptive Moment Estimation） 是一种常用的优化算法，广泛应用于深度学习中的模型训练。它结合了动量法（Momentum）和自适应学习率的优点，能够有效地加速梯度下降过程，提高训练效率和稳定性。

特点：

  1. 自适应学习率：Adam 根据每个参数的梯度信息自适应调整学习率，避免了手动调整学习率的问题。

  2. 动量：通过一阶矩的估计，Adam 具有类似动量法的特性，可以加速训练过程并减少振荡。

  3. 效率高：相比于传统的梯度下降算法，Adam 通常收敛更快，且对超参数不那么敏感。

  4. 偏差修正：Adam 通过修正初期梯度估计的偏差，使得训练过程在初期更加稳定。

 

Categorical Crossentropy 损失函数

Categorical Crossentropy（类别交叉熵）是常用于多分类问题的损失函数，特别是在神经网络的输出层采用 Softmax 激活函数时。它衡量的是模型预测的概率分布与真实标签分布之间的差异。损失越小，模型的预测越准确。

原理：

在多分类任务中，模型输出的是每个类别的概率（通过 Softmax 函数），而真实标签通常是通过 one-hot encoding 方式表示的向量（即真实类别对应的位置为 1，其他位置为 0）。

Categorical Crossentropy 计算的是真实标签和模型预测的概率之间的差异，损失值越小，说明预测结果越接近真实标签。

公式：

对于一个样本，类别交叉熵的计算公式为：

L=−∑i=1Cyilog⁡(pi)

其中：

  • C 是类别的数量。

  • yi 是实际标签的 one-hot 编码值（1 或 0）。

  • pi 是模型预测该类别的概率（由 Softmax 输出）。

  • 交叉熵是对所有类别进行求和的，真实类别的标签 yi 参与计算，其他类别的标签值为 0，因此它们对损失值没有贡献。

特点：

  1. 用于多分类任务：适用于具有多个类别的分类问题，特别是分类数量较大的情况。

  2. 适合 Softmax 输出：通常与 Softmax 激活函数一起使用，模型输出的是每个类别的概率。

  3. 概率差异度量：交叉熵量化了预测概率与真实标签之间的差异，差异越大，损失越高。

指标

在神经网络的训练过程中，loss、val_loss、acc 和 val_acc 是常见的性能指标，它们用于评估模型在训练集和验证集上的表现。下面分别解释这些指标的含义：

1. loss（损失）

  • 定义：表示训练过程中模型在训练集上的损失值。

  • 计算：损失值通常是通过模型的输出和真实标签之间的差异来计算的，具体计算方式取决于选择的损失函数。例如，对于回归问题，可能使用均方误差（MSE），对于分类问题，通常使用交叉熵（如 categoricalCrossentropy）。

  • 作用：通过不断最小化训练损失，模型的预测会变得更加精确，训练过程中我们希望看到损失值逐渐降低。

2. val_loss（验证损失）

  • 定义：表示训练过程中模型在验证集上的损失值。

  • 计算：val_loss 与 loss 的计算方法相同，区别在于它是基于验证集的预测结果计算的，而不是训练集。验证集是训练过程中未被使用的独立数据集，用来评估模型的泛化能力。

  • 作用：通过监控 val_loss，我们可以评估模型是否过拟合。如果训练损失持续下降，但验证损失开始上升，可能表明模型在训练集上过拟合。

3. acc（准确率）

  • 定义：表示模型在训练集上的准确率，即模型正确预测的样本占总样本的比例。

  • 计算：准确率是通过将预测正确的样本数除以总样本数来计算的：

    acc=正确预测的样本数总样本数

  • 作用：准确率是一个常见的衡量分类模型表现的指标，尤其是对于类别分布相对均衡的问题。在训练过程中，我们希望 acc 不断提高，表示模型在训练数据上的表现越来越好。

4. val_acc（验证准确率）

  • 定义：表示模型在验证集上的准确率。

  • 计算：val_acc 与 acc 的计算方法相同，区别在于它是基于验证集的预测结果计算的。验证集是用于评估模型泛化能力的独立数据集。

  • 作用：val_acc 可以帮助我们评估模型的泛化能力。如果 val_acc 开始停滞或下降，而训练准确率持续上升，可能表明模型在训练集上过拟合。

 

Batch，Batch Size，Epoch

Batch（批次）

  • 定义：在训练过程中，batch 指的是一次迭代中，模型处理的输入数据的子集。因为训练数据通常很大，直接将所有数据都输入模型进行训练是不切实际的，因此将数据划分成小批次来逐步训练。

  • 作用：批次的引入是为了提高训练效率，减少内存消耗并加速模型的收敛。每个批次的样本数通常小于总样本数。

  • 示例：如果训练数据包含1000个样本，且我们设置每个批次包含100个样本，那么就会有10个批次。

Batch Size（批次大小）

  • 定义：batch size 是每个批次中所包含的样本数量。它决定了每次训练过程中使用多少个样本来更新模型的参数。

  • 作用：选择合适的批次大小对训练的效果至关重要。较小的批次（如 32、64）可以让模型更频繁地更新参数，使得训练过程更灵活，但可能导致训练时间较长；较大的批次（如 128、256）可以加速训练，但可能占用更多内存，且可能导致收敛效果不如小批次。

  • 常见设置：通常，batch size 的值在 32 到 256 之间，具体选择取决于计算资源和问题本身。

  • 优点和缺点：

    • 小批次：适用于内存有限的环境，能够更快地更新模型权重，但可能导致收敛更慢或不稳定。

    • 大批次：能够利用硬件的并行处理能力，训练更高效，但可能需要更多的内存，且可能导致模型训练时不够灵活，容易陷入局部最优解。

Epoch（轮次）

  • 定义：epoch 是指整个训练数据集被完整地输入模型一次的过程。在一个 epoch 中，模型会用所有训练数据更新一次其内部的参数。

  • 作用：训练过程通常会经历多个 epoch，以便模型能够反复调整权重，逐步减小损失并提高准确率。更多的 epoch 可以帮助模型在训练集上取得更好的表现，但也可能导致过拟合，尤其是在模型训练时间过长时。

  • 示例：如果训练集有1000个样本，设置了 batch size 为 100，那么每个 epoch 需要训练 10 个批次（1000 ÷ 100 = 10）。如果训练设置了 20 个 epoch，意味着整个训练数据会被训练 20 次。

总结：

  • Batch：训练数据被分成的小块，每一块称为一个批次。

  • Batch Size：每个批次包含的样本数量。决定了每次训练时用多少数据来更新模型。

  • Epoch：整个训练集通过模型一次的过程。模型会在多个 epoch 中多次更新，逐渐学习到更好的参数。

关系：

  • 一个 epoch 包含多个 batch。

  • 训练时，数据被分成多个批次（batch），每个批次用来计算梯度并更新模型。

  • 经过多个 epochs 后，模型完成训练并得到最终的结果。

训练过程如下：

  1. 数据检查：

    • 检查数据是否加载完毕，如果没有加载完毕，函数会直接返回。

  1. 训练指标与可视化：

    • 设置了训练过程中需要跟踪的指标，包括损失（loss）、验证损失（val_loss）、准确率（acc）、验证准确率（val_acc）。

    • 设置了一个可视化容器，用于显示训练过程中的进度和指标（通过 vis.show.fitCallbacks）。

  1. 批量数据设置：

    • BATCH_SIZE：每次训练的样本数量是1000。

    • TRAIN_DATA_SIZE 和 TEST_DATA_SIZE：训练集和测试集的样本数量分别是10000和1000。

  1. 训练数据处理：

    • 通过 data.nextTrainBatch(TRAIN_DATA_SIZE) 获取训练数据，并对数据进行形状调整（28x28的灰度图像，只有一个通道）。

    • 然后对训练数据应用PCA降维（将特征维度降到50），performPCA 和 processData 是处理数据的函数。

  1. 验证数据处理：

    • 类似地，从 data.nextTestBatch(TEST_DATA_SIZE) 获取验证数据，并对验证数据进行处理（包括PCA降维）。

  1. 模型训练：

    • 使用 model.fit 方法训练模型，具体配置如下：

      • batchSize：每次训练的样本数（1000）。

      • validationData：提供验证集（测试数据）用于验证模型性能。

      • epochs：训练进行50个周期（epochs）。

      • shuffle：训练时打乱数据顺序。

      • callbacks：可视化训练过程的回调函数。

混淆矩阵

混淆矩阵 (Confusion Matrix) 是一种用于评估分类模型性能的工具，它可以直观地展示模型的预测结果与真实标签之间的关系。混淆矩阵通常用于分类任务，特别是在多分类问题中，帮助分析模型的准确性、精确度、召回率等重要指标。

结构

混淆矩阵是一个方阵，每一行表示一个真实类别，每一列表示一个预测类别。矩阵中的每个元素表示实际类别与预测类别的关系。

对于二分类问题，混淆矩阵通常如下所示：

  • True Positive (TP)：真实标签为 1，且预测为 1 的样本数量。

  • False Positive (FP)：真实标签为 0，且预测为 1 的样本数量。

  • True Negative (TN)：真实标签为 0，且预测为 0 的样本数量。

  • False Negative (FN)：真实标签为 1，且预测为 0 的样本数量。

  • 性能评估：混淆矩阵帮助我们全面了解分类模型的表现

  • 误分类分析：通过查看矩阵中的 FP 和 FN，可以发现哪些类别被误分类，以及这些误分类的模式。
